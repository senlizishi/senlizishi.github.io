Trino之实现Redis连接器支持delete操作

2021-03-16 大数据技术 Trino 评论

导读：Trino 是由 Presto SQL 更名而得，是一个分布式SQL查询引擎。Trino 目前提供了丰富的 Connectors 用于访问不同的数据源。本文接下来将讨论如何在现有的 redis connector 的基础上进行改造使其支持 delete 操作。

1、基础知识

本文只提供笔者对于 Redis Connect 支持 delete功能的大致实现思路，不做详细源码分析。朋友们需根据自身实际情况调整优化，以下代码均未在生产环境实践应用。

// 官方文档大致讲述了redis connect 的构成以及使用的方式，这里就不做细表，文档如下：
https://trino.io/docs/current/connector/redis.html
// Trino 的 github 地址，拉取后我们可以参考借鉴 Kudu 模块实现 delete 功能 
https://github.com/trinodb/trino

2、Delete 功能具体实现

2.1 重写ConnectorMetadata 3 个与 delete 有关的方法

RedisMetadata 实现了 ConnectorMetadata。 ConnectorMetadata 接口中如果实现类不重写覆盖这三个方法，则连接器本身是不支持 delete 操作的。这三个方法按被调用顺序排列如下：

  • getDeleteRowIdColumnHandle
  • beginDelete
  • finishDelete

2.1.1 先看
getDeleteRowIdColumnHandle 方法

/**
 * Get the column handle that will generate row IDs for the delete operation.
 * These IDs will be passed to the {@code deleteRows()} method of the
 * {@link io.trino.spi.connector.UpdatablePageSource} that created them.
 */
default ColumnHandle getDeleteRowIdColumnHandle(ConnectorSession session, ConnectorTableHandle tableHandle)
{
    throw new TrinoException(NOT_SUPPORTED, "This connector does not support deletes");
}

该方法目的是获取将为删除操作生成行 id 的 column handle，这些 ID 将被传递给 UpdatablePageSource#deleteRows() 。这里我参考了 KuduConnector 的做法，在 RedisColumnHandle 中创建一个固定的 ROW_ID_HANDLE。代码如下：

public final class RedisColumnHandle implements DecoderColumnHandle, Comparable<RedisColumnHandle> {
//... 此处省略其他代码
          public static final String ROW_ID_NAME = "row_id";
      public static final int ROW_ID_POSITION = -1;
      public static final RedisColumnHandle ROW_ID_HANDLE = new RedisColumnHandle(ROW_ID_POSITION, ROW_ID_NAME, VarcharType.VARCHAR, null
            , null, null, true, false, false, false);
 }

2.1.2 接下来处理 beginDelete 和 finishDelete 方法

重写 beginDelete 方法并将tableHandle转换为RedisTableHandle 返回

@Override
    public ConnectorTableHandle beginDelete(ConnectorSession session, ConnectorTableHandle tableHandle) {
        RedisTableHandle handle = (RedisTableHandle)tableHandle;
        return handle;
    }

finishDelete 则是删除后做的操作，由于删除后无后续操作，此处我就简单进行打印。

@Override
    public void finishDelete(ConnectorSession session, ConnectorTableHandle tableHandle, Collection<Slice> fragments) {
        System.out.println("finishDelete");
    }

2.2 RedisPageSourceProvider 与 RedisUpdatablePageSource

2.2.1 创建 RedisPageSourceProvider

完成了上面三个方法实现后，接下来我们需要创建一个 RedisPageSourceProvider，其实现

ConnectorPageSourceProvider，具体代码如下：

public class RedisPageSourceProvider
        implements ConnectorPageSourceProvider
{
    private final RedisRecordSetProvider recordSetProvider;

    @Inject
    public RedisPageSourceProvider(RedisRecordSetProvider recordSetProvider,RedisJedisManager jedisManager)
    {
        this.recordSetProvider = requireNonNull(recordSetProvider, "recordSetProvider is null");
    }

    @Override
    public ConnectorPageSource createPageSource(
            ConnectorTransactionHandle transaction,
            ConnectorSession session,
            ConnectorSplit split,
            ConnectorTableHandle table,
            List<ColumnHandle> columns,
            TupleDomain<ColumnHandle> dynamicFilter)
    {
        RedisRecordSet recordSet = (RedisRecordSet) recordSetProvider.getRecordSet(transaction, session, split, table, columns);
       // 此处为关键
      if (columns.contains(RedisColumnHandle.ROW_ID_HANDLE)) {
            return new RedisUpdatablePageSource(recordSet,recordSetProvider.getJedisManager());
        }
        return new RecordPageSource(recordSet);
    }
}

判断列组中如果包含我们前面创建的 ROW_ID_HANDLE，则会创建一个新的 RedisUpdatablePageSource 对象，该对象内部可编写具体的删除处理逻辑。因此在创建RedisUpdatablePageSource 时我们将 JedisManager 一并传入，用于与 redis 交互。

2.2.2 注册 RedisPageSourceProvider

这里与上一篇 insert 操作创建 RedisPageSinkProvider 流程一致，我们也需要在 RedisConnectorModule 中将我们刚创建的 RedisPageSourceProvider 绑定注册。

public class RedisConnectorModule implements Module {
  binder.bind(RedisPageSourceProvider.class).in(Scopes.SINGLETON);
}

在 RedisConnector 类中新增 redisPageSourceProvider 属性，并提供 get 方法。

public class RedisConnector implements Connector {
   // 此处省略部分代码
  private final RedisPageSourceProvider redisPageSourceProvider;
  
  @Override
    public ConnectorPageSourceProvider getPageSourceProvider() {
        return redisPageSourceProvider;
    }
}

这里有一个注意点，原本的 redis 连接器本身内置并注册了 RedisRecordSetProvider。在运行时有可能出现 Connector ‘redis’ returned both page source and record set providers 异常。追踪源码发现：

// ConnectorManager 的内部类 MaterializedConnector 中有这么一段代码
ConnectorPageSourceProvider connectorPageSourceProvider = null;
            try {
                connectorPageSourceProvider = connector.getPageSourceProvider();
                requireNonNull(connectorPageSourceProvider, format("Connector '%s' returned a null page source provider", catalogName));
            }
            catch (UnsupportedOperationException ignored) {
            }

            try {
                ConnectorRecordSetProvider connectorRecordSetProvider = connector.getRecordSetProvider();
                requireNonNull(connectorRecordSetProvider, format("Connector '%s' returned a null record set provider", catalogName));
                verify(connectorPageSourceProvider == null, "Connector '%s' returned both page source and record set providers", catalogName);
                connectorPageSourceProvider = new RecordPageSourceProvider(connectorRecordSetProvider);
            }
            catch (UnsupportedOperationException ignored) {
            }
            this.pageSourceProvider = Optional.ofNullable(connectorPageSourceProvider);

当连接器的
connectorPageSourceProvider 和
connectorRecordSetProvider 同时不为空时就会报出该异常。解决的方式只需改动如下一处地方的代码：

// 将 RedisConnector 类中的 getRecordSetProvider 修改为抛出UnsupportedOperationException
@Override
    public ConnectorRecordSetProvider getRecordSetProvider() {
        throw new UnsupportedOperationException();
    }

2.2.3 创建 RedisUpdatablePageSource

这里核心的删除逻辑写在 deleteRows() 中，关于 Page、Block 和 Slice 等概念可参考文章Presto核心数据结构：Slice Block Page

public class RedisUpdatablePageSource
        implements UpdatablePageSource {
    private final RecordPageSource inner;
    private final Jedis jedis;

    Logger log = Logger.get(RedisPageSink.class);
   
   // 初始化
    public RedisUpdatablePageSource(RedisRecordSet recordSet, RedisJedisManager jedisManager) {
        this.inner = new RecordPageSource(recordSet);
        RedisConnectorConfig redisConnectorConfig = jedisManager.getRedisConnectorConfig();
        Set<HostAddress> hostAddressSet = redisConnectorConfig.getNodes();
        HostAddress hostAddress;
        if (hostAddressSet.size() > 0) {
            hostAddress = new ArrayList<>(redisConnectorConfig.getNodes()).get(0);
            log.info("当前连接的 redis 地址为：" + hostAddress.getHostText() + ":" + hostAddress.getPort());
        } else {
            throw new RuntimeException("中获取不到 redis 节点的地址信息");
        }
        JedisPool jedisPool = jedisManager.getJedisPool(HostAddress.fromParts(hostAddress.getHostText(), hostAddress.getPort()));
        this.jedis = jedisPool.getResource();
    }
        
   // 删除逻辑
    @Override
    public void deleteRows(Block rowIds) {
        for (int i = 0; i < rowIds.getPositionCount(); i++) {
            int len = rowIds.getSliceLength(i);
            Slice slice = rowIds.getSlice(i, 0, len);
            String deleteKey = new String(slice.getBytes());
            // 删除键
            jedis.del(deleteKey);
        }
    }

    @Override
    public void close() {
       // 关闭连接
        jedis.close();
        inner.close();
    }

//.. 此处省略部分代码
}
