Calcite之自定义函数

2021-01-21 大数据技术 Calcite 评论

导读：本文将通过一个简单例子讲述如何在 Calcite 中注册自定义函数（UDF）并应用。

编写一个简单的 UDF

/**
 * UDF 在字符串末端添加 test 字符
 */
public class SimpleFun {
    public String eval(String data) {
        return data + "test";
    }
}

编写 UDF 测试类

/**
  * UDF 测试类
 */
public class UDFTest {

    final static Logger logger = LoggerFactory.getLogger(UDFTest.class);
    final static TablePrinter printer = TablePrinter.DEFAULT;
    private CalciteConnection conn;

    @Before
    public void setup() throws SQLException, ClassNotFoundException {
        // 驱动配置
        Class.forName("org.apache.calcite.jdbc.Driver");
        Properties config = new Properties();
        config.put("model", UDFTest.class.getClassLoader().getResource("schema.json").getPath());
        config.put("lex", "MYSQL");
        config.put("caseSensitive", "false");
        // 获取 CalciteConnection 连接
        conn = (CalciteConnection)DriverManager.getConnection("jdbc:calcite:", config);
        SchemaPlus plus = conn.getRootSchema();
        // 注册 UDF
        plus.add("addString", ScalarFunctionImpl.create(SimpleFun.class, "eval"));
    }

    /**
     * 测试
     */
    @Test
    public void udfTest() throws Exception {
        try (Statement stmt = conn.createStatement()) {
            List rs = ZDCalciteUtils.resultSetToList(stmt.executeQuery(
                    "select addString(name) from flink_k8s.k8s_test"
            ));
            logger.info("结果集: \n{}", printer.render(rs));
        }
    }

    @After
    public void shutdown() throws SQLException {
        // 关闭连接
        conn.close();
    }
}

解析

Demo 中通过调用 CalciteConnection 的 getRootSchema() 方法获取到 SchemaPlus。通过 SchemaPlus 的 add() 方法实现 UDF 注册，之后便可以在 SQL 语句中应用。

接下来，我们进一步深入分析其是如何进行UDF注册的。通过进行断点跟踪发现在
CalcitePrepareImpl.createSqlValidator() 方法中出现两个重要的对象

  1. CalciteCatalogReader
  2. SqlOperatorTable
private SqlValidator createSqlValidator(Context context,CalciteCatalogReader catalogReader) {
        // 获取 SqlStdOperatorTable.instance()
        final SqlOperatorTable opTab0 =
        context.config().fun(SqlOperatorTable.class,
                SqlStdOperatorTable.instance());
        // 整合SqlOperatorTable
        final SqlOperatorTable opTab =
                ChainedSqlOperatorTable.of(opTab0, catalogReader);
        final JavaTypeFactory typeFactory = context.getTypeFactory();
        final SqlConformance conformance = context.config().conformance();
        return new CalciteSqlValidator(opTab, catalogReader, typeFactory,
                conformance);
    }

1、SqlOperatorTable

其主要定义了一个用于枚举和查找SQL运算符和函数的接口。

public interface SqlOperatorTable {
  
  /**
   * Retrieves a list of operators with a given name and syntax. For example,
   * by passing SqlSyntax.Function, the returned list is narrowed to only
   * matching SqlFunction objects.
   *
   * @param opName   name of operator
   * @param category function category to look up, or null for any matching
   *                 operator
   * @param syntax   syntax type of operator
   * @param operatorList mutable list to which to append matches
   * @param nameMatcher Name matcher
   */
   void lookupOperatorOverloads(SqlIdentifier opName,
      @Nullable SqlFunctionCategory category,
      SqlSyntax syntax,
      List<SqlOperator> operatorList,
      SqlNameMatcher nameMatcher);
}

2、CatalogReader

其继承了 SqlOperatorTable

public interface CatalogReader extends RelOptSchema, SqlValidatorCatalogReader, SqlOperatorTable {
        ThreadLocal<Prepare.CatalogReader> THREAD_LOCAL = new ThreadLocal();

        Prepare.PreparingTable getTableForMember(List<String> var1);

        Prepare.CatalogReader withSchemaPath(List<String> var1);

        Prepare.PreparingTable getTable(List<String> var1);
    }

我们注册的自定义的UDF信息存于
CatalogReader.rootSchema.functionMap 中。

流程总结

  • 我们获取到SchemaPlus调用 add() 进行UDF注册，其将UDF信息保存到 CatalogReader.rootSchema.functionMap 中
  • Calcite 在parse的过程中会先根据函数名调用SqlStdOperatorTable 的 lookupOperatorOverloads方法查找对应函数，如果没有找到，会自动将函数解析成SqlUnresolvedFunction。
  • 在 validate 过程会将多个注册表chain起来，并根据函数名在 ChainedSqlOperatorTable中查找函数，之后进行验证。
