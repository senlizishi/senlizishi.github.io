Trino之核心数据结构：Slice Block Page

2021-03-10 大数据技术 Trino 评论

本文翻译自：
https://zhuanlan.zhihu.com/p/60813087

导读：在Presto中，我们需要了解一些非常重要的数据结构，Slice、Block 和 Page是其中的三个，我将在本文中介绍这些数据结构。

Slice

从用户的角度来看，Slice是一个对开发者更友好的虚拟内存，它定义了一组getter和setter，所以你可以像使用结构化数据一样使用内存:

Slice的典型用法是表示字符串：

// use it as utf8 encoded string
Slice slice = Slices.utf8Slice("hello");
Slice subSlice = SliceUtf8.substring(slice, 1, 2);

你可以看到我们像使用字符串一样使用Slice, Presto更喜欢Slice而不是String的原因是:

  • 字符串的构建成本很高（String concatenation, StringBuilder 等）
  • Slice是可变的，而String不是，所以当我们需要进行字符串计算时，Slice的效率更高
  • 字符串在内存中编码为UTF16，而Slice使用UTF8，这在内存中效率更高
  • UTF16使用最少两个字节来表示一个字符，而UTF8使用最少一个字节，所以如果字符串内容主要是ascii字符，UTF8可以节省大量内存

另外一种使用 Slice 的方法（在 Presto 中 ）是用于表示 raw bytes (SQL 中的 VARBINARY 类型 )：

// use it as raw bytes
block.getSlice().getBytes()

你可以从一个 Block 中获取一个 Slice 就像原始类型Int, Long等。

Block

Page 是由 Block 组成，因此我们先介绍 Block。Block 可以看成一个array，array 中的数据都具有相同类型（int,long,Slice等）。每个数据都占据着一个 position，总 position 数表示该 Block 持有的数据的总行数（Block 只保存这些行中的一列）

Block 定义了一些 APIs，其中之一是 getXXX 方法，让我们以 getInt 作为例子：

/**
    * Gets a little endian int at {@code offset} in the value at {@code position}.
    */
default int getInt(int position, int offset)
{
    throw new UnsupportedOperationException(getClass().getName());
}

通常，一个块只支持一个getXxx方法，因为一个 Block 中的所有数据都属于一个具有某种类型的列。

Block定义的另一个方法是copyPositions，它不是从Block中获取一个值，而是获取一个由位置列表指定的值列表作为一个新块，我猜想这在映射中使用。

/**
 * Returns a block containing the specified positions.
 * All specified positions must be valid for this block.
 * <p>
 * The returned block must be a compact representation of the original block.
 */
Block copyPositions(List<Integer> positions);

除了Block, Presto还定义了BlockEncoding，它决定了Block如何序列化和反序列化:

public interface BlockEncoding
{
    ...

    /**
     * Read a block from the specified input.  The returned
     * block should begin at the specified position.
     */
    Block readBlock(SliceInput input);

    /**
     * Write the specified block to the specified output
     */
    void writeBlock(SliceOutput sliceOutput, Block block);
    ...
}

让我们使用一个最简单的 BlockEncoding : IntArrayBlockEncoding 作为例子，它的 readBlock 看起来是这样的：

int positionCount = block.getPositionCount();
sliceOutput.appendInt(positionCount);

encodeNullsAsBits(sliceOutput, block);

for (int position = 0; position < positionCount; position++) {
    if (!block.isNull(position)) {
        sliceOutput.writeInt(block.getInt(position, 0));
    }
}

就像我们从代码中所看到的，它首先对 position 进行编码，然后将数据isNull编码为位图，最后把所有的整数都写进输出。看起来与网络协议代码非常相似。

Page

Page 是由块组成

public class Page
{
    private final Block[] blocks;
    private final int positionCount;
    ...
}

除了 Blocks，Page 还有另外一个概念叫做 channel：每一个 Block 都是 Page 的一个 channel，blocks 的总数就是 channel 的总数。这里让我们做一下总结，当有一些 rows 要被发送时，Presto 将会

  • 把每一列放在单独的 Block 中
  • 把这些 Block 放到一个 Page 里
  • 发送 Page

Page是一种数据结构，它保存数据并在 Presto 物理执行操作符之间传输: upstream 操作符产生输出通过 getOutput():

/**
 * Gets an output page from the operator.  If no output data is currently
 * available, return null.
 */
Page getOutput();

Downstream 操作符获得输入通过 addInput() 方法：

/**
 * Adds an input page to the operator.  This method will only be called if
 * {@code needsInput()} returns true.
 */
void addInput(Page page);

和Block一样，Page也需要序列化和反序列化，序列化发生在数据需要在worker之间传输的时候，当Page被序列化时，它将首先使用相应的BlockEncoding对块进行编码，接着如果一个压缩器（compressor ）可用，它将尝试压缩已编码的块数据，如果压缩工作良好(编码率低于:0.8)，它将使用压缩后的数据，否则使用解压缩数据。编码后的块数据将与一些统计信息一起放入一个名为SerializedPage的类中:压缩前和压缩后的页面字节大小。

总结

今天我们介绍了Presto的三种核心数据结构:Slice、Block 和Page。简而言之，Slice是更便于开发人员使用的虚拟内存，块表示一个列，页表示一个行组。
